<workflow_spec>
Contenu canonique de `.github/workflows/ci.yml` :

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

# Cancel in-progress runs on the same branch when a new push arrives.
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:

  gates:
    name: Lint / Licence / Headers / Deps
    runs-on: ubuntu-latest
    timeout-minutes: 20
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install Flutter
        uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.41.5'
          channel: stable
          cache: true

      - name: Pub get
        run: flutter pub get

      - name: Dart format check
        run: dart format --set-exit-if-changed .

      - name: Flutter analyze
        run: flutter analyze --fatal-infos --fatal-warnings

      - name: Check GOSL headers
        run: dart run tool/check_headers.dart

      - name: Check licenses (GPL/AGPL/copyleft scan)
        run: dart run tool/check_licenses.dart

      - name: Check DEPENDENCIES.md is up to date
        run: dart run tool/check_dependencies_md.dart

      - name: Tool scripts unit tests
        run: dart test tool/test/

      - name: Flutter unit + widget tests
        run: flutter test

      - name: Flutter test with DEBUG define
        run: flutter test --dart-define=DEBUG=true test/file_logger_debug_define_test.dart

  android:
    name: Build Android APK (debug)
    needs: gates
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup JDK 17
        uses: actions/setup-java@v4
        with:
          distribution: 'temurin'
          java-version: '17'

      - name: Install Flutter
        uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.41.5'
          channel: stable
          cache: true

      - name: Pub get
        run: flutter pub get

      - name: Build APK (debug)
        run: flutter build apk --debug

      - name: Upload APK artifact
        uses: actions/upload-artifact@v4
        with:
          name: mirkfall-android-debug-apk
          path: build/app/outputs/flutter-apk/app-debug.apk
          if-no-files-found: error
          retention-days: 14

  ios:
    name: Build iOS (no-codesign)
    needs: gates
    # Pin runner per RESEARCH.md §Pitfall 3 (macos-latest auto-rolls)
    runs-on: macos-14
    timeout-minutes: 45
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Pin Xcode version
        uses: maxim-lobanov/setup-xcode@v1
        with:
          xcode-version: '16.1'

      - name: Install Flutter
        uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.41.5'
          channel: stable
          cache: true

      - name: Pub get
        run: flutter pub get

      - name: CocoaPods install
        working-directory: ios
        run: pod install

      - name: Build iOS (no-codesign)
        run: flutter build ios --release --no-codesign
      # No IPA artifact upload in Phase 01. Phase 15 (release) adds signed IPA.
```

**Décisions documentées :**

- **Job order :** `gates` bloque `android` + `ios` via `needs: gates`. Si les gates échouent, les builds ne tournent même pas. Économie de CI minutes + feedback rapide.
- **Concurrency group :** un push tue les runs précédents de la même branche. Évite les minutes gaspillées pendant l'itération.
- **Timeouts :** `gates` 20 min (tout sauf build, sandbox sans Java), `android` 30 min (gradle pre-warm long au premier run), `ios` 45 min (pod install + xcode premier run).
- **macos-14 + Xcode 16.1 :** pinné pour éviter la dérive runner image (RESEARCH.md §Pitfall 3). `macos-14` est disponible en 2026. Si 2027 force un bump, la CI fail rapidement et on update.
- **Java 17 :** requis par Android Gradle Plugin 8.x (embedded dans Flutter 3.41).
- **`cache: true` on subosito/flutter-action :** cache pub + Gradle automatiquement. Gain typique 2-3 min par run.
- **APK debug, pas release :** Phase 01 n'a pas de signing keys. `--debug` tourne sans config.
- **Pas d'artifact IPA :** iOS build réussi = compile OK. Sideload via CI + artifact = Phase 15 (release flow).
- **Pas d'upload APK dans le job gates :** séparation des responsabilités. Gates = audit, build jobs = artifact.

**Note sur l'audit des GitHub Actions externes :** `subosito/flutter-action@v2`, `maxim-lobanov/setup-xcode@v1`, `actions/setup-java@v4`, `actions/checkout@v4`, `actions/upload-artifact@v4` — toutes MIT, déjà listées dans `DEPENDENCIES.md` §Tooling / GitHub Actions (Plan 03). **Important :** à chaque major bump (ex: `@v4` → `@v5`), re-auditer la licence et vérifier l'absence de télémétrie nouvelle. À tracer dans un futur commit `DEPENDENCIES.md`.

Si un job échoue, copier les logs de l'étape qui fail et revenir pour ouvrir un plan de gap closure :
