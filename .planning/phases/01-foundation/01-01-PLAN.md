RESEARCH.md §Open Question 1 a flaggé que la variante moderne (`PlatformDispatcher.onError`) est préférée par la communauté depuis Flutter 3.10, mais on respecte CONTEXT.md ; si l'utilisateur change d'avis plus tard, la migration est locale à `main.dart`.

```dart
// Copyright (c) 2026 THONGVAN Alexis
// Licensed under the Good Old Software License v1.0
// See LICENSE file for details

import 'dart:async';
import 'package:flutter/foundation.dart';
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:logging/logging.dart';

import 'app.dart';

Future<void> main() async {
  await runZonedGuarded<Future<void>>(() async {
    WidgetsFlutterBinding.ensureInitialized();

    // Phase 02 branchera le FileLogger ici. Phase 01 se contente de
    // configurer un handler minimal qui print vers la console en debug.
    Logger.root.level = Level.INFO;
    Logger.root.onRecord.listen((LogRecord rec) {
      if (kDebugMode) {
        // ignore: avoid_print
        debugPrint('[${rec.level.name}] ${rec.loggerName}: ${rec.message}');
      }
    });

    final log = Logger('main');

    FlutterError.onError = (FlutterErrorDetails details) {
      log.shout('FlutterError', details.exception, details.stack);
      if (kDebugMode) {
        FlutterError.dumpErrorToConsole(details);
      }
    };

    runApp(const ProviderScope(child: MirkFallApp()));
  }, (Object error, StackTrace stack) {
    Logger('main').shout('uncaughtZoneError', error, stack);
  });
}
```

NOTE workaround : `debugPrint` est utilisé en lieu et place de `print` dans le handler `Logger.root.onRecord` de Phase 01 car `avoid_print` est `error` dans `analysis_options.yaml`. Phase 02 remplace ce listener par `FileLogger.bootstrap()` qui écrit en JSONL.
