<algorithm>
From 09-RESEARCH §computeRevealMask Algorithm Specification — bbox-first + per-cell Haversine:

```
1. Compute parent tile bbox (NW, SE) via TileMath.tileToLatLon(parentX, parentY, parentZoom) and (parentX+1, parentY+1, parentZoom).
2. Compute circle bbox: expand (centerLat, centerLon) by ±radiusMeters in each direction (crude Mercator: 1° lat ≈ 111_320 m; 1° lon ≈ 111_320 * cos(lat) m).
3. If circle bbox does NOT intersect parent tile bbox → return all-zero mask (early exit).
4. Compute cell spans: cellLatSpan = (parentNw.lat - parentSe.lat) / 64 ; cellLonSpan = (parentSe.lon - parentNw.lon) / 64.
5. Compute cell index range (cellJStart..cellJEnd, cellIStart..cellIEnd) from circle bbox clipped to parent tile.
6. For each (i, j) in the range:
   a. Compute cell rectangle corners (4 lat/lons).
   b. Compute "closest point on cell to circle centre" via axis-clamp (this is the test for rect-circle intersection).
   c. Compute Haversine distance from circle centre to closest point.
   d. If distance ≤ radiusMeters → flip bit (j * 64 + i).
7. Return the 512-byte mask.
```

Helpers needed (private, top-of-file):
- `_circleBbox(lat, lon, r)` → (minLat, maxLat, minLon, maxLon)
- `_bboxesIntersect(a, b)` → bool
- `_haversineMeters(lat1, lon1, lat2, lon2)` → double
- `_clamp(value, low, high)` → double

Acceptable to use `dart:math` (log, sin, cos, atan2, sqrt, pi). NOT acceptable: importing Flutter, Drift, or any infrastructure layer.

  <behavior>
    Given (centerLat, centerLon, radiusMeters, parentX, parentY, parentZoom=14):
    - Returns 512-byte Uint8List.
    - For each cell (i, j) in the 64×64 grid of the parent tile:
      - If the cell rectangle intersects the circle → bit (j*64+i) is 1.
      - Otherwise → bit is 0.
    - "Intersects" means "minimum distance from circle centre to cell rectangle ≤ radius" (not "centre inside circle"). This is the MIRK-03 no-micro-holes decision from CONTEXT.
    - Metre accuracy: uses Haversine on the closest rect-corner-or-edge point, not lat/lon Euclidean.
    - Returns all-zero mask if circle bbox does not intersect parent tile bbox.
    - Returns all-zero mask if radiusMeters ≤ 0 (defensive).

    Test cases:
    1. **Circle fully inside parent tile, centre (45.0, 5.0), r=25m** — popcount ~20-25 bits; bits form a disc shape.
    2. **Circle fully outside parent tile** — all-zero bitmap.
    3. **Circle straddling a parent-tile east-west boundary** — calling twice (once per parent tile) yields two non-empty bitmaps; union covers the full disc.
    4. **Circle straddling a north-south boundary** — symmetric to (3).
    5. **Tiny radius (1m)** — at most 1-2 cells flipped.
    6. **Large radius (120m, covering ≥ half the parent tile at z=14)** — many bits set; inner cells (far from edge) are guaranteed set.
    7. **Fix at the exact parent tile corner** — reveal touches all 4 parent tiles in a neighbourhood (parent + N, E, NE neighbours); plan 09-03 tests 2 of the 4 via parameterized test.
    8. **Polar clamp**: fix at lat=85.0, lon=5.0 — TileMath clamps; mask is still valid (all-zero or mostly-zero depending on r).
    9. **Antimeridian**: not tested (25 m never crosses ±180°). Documented as out-of-scope.
    10. **Radius ≤ 0** — returns all-zero (defensive).
  </behavior>

```dart
// lib/domain/revealed/reveal_calculator.dart — add after Phase 03 mergeBitmap/popcount

import 'dart:math' as math;
import 'dart:typed_data';
import 'tile_math.dart';
import 'revealed_tile.dart';  // for kRevealedTileBitmapBytes + kRevealedTileBitmapCellsPerSide

Uint8List computeRevealMask({
  required double centerLat,
  required double centerLon,
  required double radiusMeters,
  required int parentX,
  required int parentY,
  required int parentZoom,
}) {
  final mask = Uint8List(kRevealedTileBitmapBytes);
  if (radiusMeters <= 0.0) return mask;  // defensive — no-op

  // 1. Parent tile bbox from TileMath (NW + SE corners)
  final parentNw = TileMath.tileToLatLon(x: parentX, y: parentY, zoom: parentZoom);
  final parentSe = TileMath.tileToLatLon(
      x: parentX + 1, y: parentY + 1, zoom: parentZoom);
  // Parent tile = lat ∈ [parentSe.lat, parentNw.lat], lon ∈ [parentNw.lon, parentSe.lon]

  // 2. Circle bbox (Mercator-approximated degree extents at centre latitude)
  final latDegPerMetre = 1.0 / _metresPerDegreeLat;
  final lonDegPerMetre = 1.0 / (_metresPerDegreeLat * math.cos(_toRad(centerLat)));
  final circleMinLat = centerLat - radiusMeters * latDegPerMetre;
  final circleMaxLat = centerLat + radiusMeters * latDegPerMetre;
  final circleMinLon = centerLon - radiusMeters * lonDegPerMetre;
  final circleMaxLon = centerLon + radiusMeters * lonDegPerMetre;

  // 3. Early exit if circle bbox outside parent bbox
  if (circleMaxLat < parentSe.lat || circleMinLat > parentNw.lat ||
      circleMaxLon < parentNw.lon || circleMinLon > parentSe.lon) {
    return mask;
  }

  // 4. Cell span + clipped cell index range
  const cellsPerSide = kRevealedTileBitmapCellsPerSide;  // 64
  final cellLatSpan = (parentNw.lat - parentSe.lat) / cellsPerSide;
  final cellLonSpan = (parentSe.lon - parentNw.lon) / cellsPerSide;

  final cellJStart = math.max(
      0, ((parentNw.lat - circleMaxLat) / cellLatSpan).floor());
  final cellJEnd = math.min(
      cellsPerSide - 1, ((parentNw.lat - circleMinLat) / cellLatSpan).ceil());
  final cellIStart = math.max(
      0, ((circleMinLon - parentNw.lon) / cellLonSpan).floor());
  final cellIEnd = math.min(
      cellsPerSide - 1, ((circleMaxLon - parentNw.lon) / cellLonSpan).ceil());

  // 5. Per-cell geometric intersect (Haversine-accurate)
  for (var j = cellJStart; j <= cellJEnd; j++) {
    final cellNorthLat = parentNw.lat - j * cellLatSpan;
    final cellSouthLat = parentNw.lat - (j + 1) * cellLatSpan;
    for (var i = cellIStart; i <= cellIEnd; i++) {
      final cellWestLon = parentNw.lon + i * cellLonSpan;
      final cellEastLon = parentNw.lon + (i + 1) * cellLonSpan;

      // Closest point on cell rectangle to circle centre (axis-clamp)
      final closestLat = _clamp(centerLat, cellSouthLat, cellNorthLat);
      final closestLon = _clamp(centerLon, cellWestLon, cellEastLon);

      final d = _haversineMeters(centerLat, centerLon, closestLat, closestLon);
      if (d <= radiusMeters) {
        final bitIndex = j * cellsPerSide + i;
        final byteIndex = bitIndex >> 3;
        final bitOffset = bitIndex & 7;
        mask[byteIndex] |= 1 << bitOffset;
      }
    }
  }
  return mask;
}

// ---- Helpers (private, top-level) ----

const double _metresPerDegreeLat = 111_320.0;  // Approximate; good to 0.5 % globally
const double _earthRadiusMeters = 6_371_008.8;  // WGS-84 mean radius

double _toRad(double deg) => deg * math.pi / 180.0;

double _clamp(double value, double low, double high) {
  if (value < low) return low;
  if (value > high) return high;
  return value;
}

/// Great-circle distance in metres via the Haversine formula.
double _haversineMeters(double lat1, double lon1, double lat2, double lon2) {
  final dLat = _toRad(lat2 - lat1);
  final dLon = _toRad(lon2 - lon1);
  final l1 = _toRad(lat1);
  final l2 = _toRad(lat2);
  final a = math.sin(dLat / 2) * math.sin(dLat / 2) +
            math.sin(dLon / 2) * math.sin(dLon / 2) * math.cos(l1) * math.cos(l2);
  final c = 2 * math.atan2(math.sqrt(a), math.sqrt(1 - a));
  return _earthRadiusMeters * c;
}
```
