<seeding_strategy>
First-launch seeding of the 4 builtin MirkStyle rows:

**Decision**: lazy seed inside `builtinMirkStylesProvider.build()`. Behavior:
1. Provider reads `ref.watch(mirkStyleStoreProvider)`.
2. Calls `store.list()`.
3. For each `kBuiltinMirkStyles` descriptor not yet present in the list, calls `store.insert(MirkStyle(id: ..., config: descriptor.defaultConfig(), ...))`.
4. Returns the final list of `MirkStyle` for the 4 builtins (whether pre-existing or just inserted).

**Why lazy vs main.dart bootstrap**:
- Main.dart bootstrap is the Phase 07 `FirstLaunchWorldCopier` pattern ‚Äî works but needs a new step in `main()`.
- Lazy provider seeding keeps Phase 09 self-contained; no bootstrap edits.
- Downside: a consumer must read the provider for seeding to happen. Plan 09-07 wires the picker sheet which will naturally read it.
- Idempotence: the `for each not-yet-present` check makes repeated reads safe.

**MirkStyleId generation for builtins**: deterministic ‚Äî `MirkStyleId('style_builtin_atmospheric')`, etc. Not ULID. The ID doubles as a marker of "built-in" at the DB layer (user imports get real ULIDs). This is worth a docstring in `MirkStyle` entity if it's load-bearing for Phase 13 (delete-if-not-builtin semantics in OPT-04).
</seeding_strategy>

/// Resolves a [MirkStyleConfig] to its concrete [MirkRenderer].
///
/// The switch is exhaustive at compile time ‚Äî adding a new sealed
/// variant to [MirkStyleConfig] triggers an "missing case" analyzer
/// error here. This is the load-bearing structural enforcement for
/// MIRK-05 "adding a style = editing 3 core files" ‚Äî the sealed union,
/// the registry constant, and THIS factory.
///
/// Fallback semantics:
/// * [ShaderConfig] ‚Äî Phase 13 body. Returns a [ShaderMirkRenderer] stub
///   whose paint() throws UnimplementedError. Session consumers that
///   reach this branch in Phase 09 have a configuration bug (no user
///   action path leads to it in the V1.0 scope).
/// * [UnknownConfig] ‚Äî cross-version forward-compat fallback (Phase 03).
///   The local app cannot render what it does not understand, so we
///   degrade to the default atmospheric experience. Logged at creation
///   time via `Logger('infrastructure.mirk.factory')`.
class MirkRendererFactory {
  const MirkRendererFactory();

  MirkRenderer create(MirkStyleConfig config) {
    return switch (config) {
      AtmosphericConfig() => AtmosphericMirkRenderer(config),
      SolidConfig() => SolidFillMirkRenderer(config),
      CandlelightConfig() => CandlelightMirkRenderer(config),
      HeavenlyCloudsConfig() => HeavenlyCloudsMirkRenderer(config),
      ShaderConfig() => ShaderMirkRenderer(config),
      UnknownConfig() => AtmosphericMirkRenderer(const AtmosphericConfig()),
    };
  }
}

import '../../domain/mirk/mirk_style.dart';

/// Returns the 4 built-in [MirkStyle] entities, lazily seeding missing
/// rows into `t_mirk_styles` on first read.
///
/// Seed semantics:
/// * Idempotent ‚Äî reading a second time is a pure DB read (no inserts).
/// * Safe against manual DB deletion ‚Äî if a builtin row is gone, next
///   read re-inserts it.
/// * Deterministic IDs ‚Äî `kBuiltinMirkStyles[i].id` is used verbatim
///   (not a ULID) so future OPT-04 code can mark built-ins as
///   non-deletable by prefix match on `style_builtin_`.
///
/// `keepAlive: true` ‚Äî styles list rarely changes; avoids repeated DB
/// read cost on every consumer rebuild.
@Riverpod(keepAlive: true)
Future<List<MirkStyle>> builtinMirkStyles(Ref ref) async {
  final store = ref.watch(mirkStyleStoreProvider);
  final existing = await store.list();
  final byId = {for (final s in existing) s.id.value: s};
  final result = <MirkStyle>[];
  for (final descriptor in kBuiltinMirkStyles) {
    final existing = byId[descriptor.id];
    if (existing != null) {
      result.add(existing);
      continue;
    }
    final fresh = MirkStyle(
      id: MirkStyleId(descriptor.id),
      displayName: descriptor.displayName,
      config: descriptor.defaultConfig(),
      createdAtUtc: DateTime.now().toUtc(),
      createdAtOffsetMinutes: DateTime.now().timeZoneOffset.inMinutes,
    );
    await store.insert(fresh);
    result.add(fresh);
  }
  return result;
}

/// Resolves the currently-active MirkRenderer for the active session.
///
/// Lifecycle:
/// * `ref.watch(activeSessionControllerProvider)` ‚Äî changes invalidate
///   the renderer when the active session changes.
/// * Returns `NoopMirkRenderer` when no session is active.
/// * Returns `AtmosphericMirkRenderer(default)` when the session has no
///   `mirk_style_id` or the referenced style is missing / UnknownConfig.
/// * `ref.onDispose` calls `renderer.dispose()` when the provider is
///   torn down (session change, style swap, app exit). This ensures
///   GPU resources + tickers + noise generators are released.
///
/// `keepAlive: false` ‚Äî the renderer is session-scoped and should be
/// GC'd when the session ends or the style changes.
@riverpod
Future<MirkRenderer> activeMirkRenderer(Ref ref) async {
  final sessionState = ref.watch(activeSessionControllerProvider);
  final factory = ref.watch(mirkRendererFactoryProvider);
  final styleStore = ref.watch(mirkStyleStoreProvider);

  // Extract active session (sealed ActiveSessionState ‚Äî pattern match)
  final activeSession = sessionState.valueOrNull?.whenOrNull(
    tracking: (session, _) => session,
    // other states (idle, error, etc.) return null
  );

  if (activeSession == null) {
    final noop = const NoopMirkRenderer();
    ref.onDispose(() => noop.dispose());
    return noop;
  }

  MirkStyleConfig config = const MirkStyleConfig.atmospheric();
  if (activeSession.mirkStyleId case final styleId?) {
    final style = await styleStore.findById(styleId);
    if (style != null) {
      config = style.config;
      // UnknownConfig still goes to factory ‚Äî factory returns atmospheric as fallback
    }
  }

  final renderer = factory.create(config);
  ref.onDispose(() => renderer.dispose());
  return renderer;
}
