<fixture_design>
Per 09-RESEARCH §Fixture 50k Strategy + Format:

- Source of truth: `tool/fixtures/build_50k_tiles.dart` deterministic builder.
- Committed artefact: `test/fixtures/mirk/fifty_k_tiles_seed.sql` — text SQL with 50,000 INSERT statements.
- Integrity gate: `tool/check_mirk_fixture_fresh.dart` runs the builder to a tmp file and byte-compares.

Builder parameters (hardcoded constants in the builder file):
- Session: `id = 'sess_01FIFTYKTEST0000000000000'`
- Tile grid: 500 × 100 starting at (parentX=8400, parentY=5500), parentZoom=14
- Bitmap density: ~25% bits set (1024 of 4096), seeded by `math.Random(0x4D49524B)` ("MIRK")
- `set_bit_count`: exact popcount of each bitmap
- `updated_at_utc`: fixed ISO string `'2026-01-01T00:00:00.000Z'`

Size budget: 50,000 × ~120 bytes/row hex-encoded ≈ 6-8 MB text. Commit to git. If > 20 MB, switch to `.sql.gz` with decompression at load time.

Hex-encoded bitmap: SQLite/Drift `X'...'` hex literal. Each INSERT:
```sql
INSERT INTO t_revealed_tiles (id, session_id, parent_x, parent_y, parent_zoom, bitmap, set_bit_count, updated_at_utc)
VALUES ('rtil_01FIFTY00001', 'sess_01FIFTYKTEST0000000000000', 8400, 5500, 14, X'A1B2...', 1024, '2026-01-01T00:00:00.000Z');
```

Schema prerequisites: load order starts with the `t_sessions` row, then 50k `t_revealed_tiles` rows.
</fixture_design>

  /// viewport-sized subset is loaded.
  void seed1000TilesEurope({required SessionId sessionId}) {
    final rng = math.Random(0xE05EED);
    for (var i = 0; i < 1000; i++) {
      final lat = 43.0 + rng.nextDouble() * 7.0;
      final lon = 0.0 + rng.nextDouble() * 15.0;

        sessionId: sessionId,
        parentX: tile.x, parentY: tile.y, parentZoom: 14,
        bitmap: bitmap,

      );
    }
