**Atomic commit format:**
```
<type>(02-rev): <title from triage table>

Addresses [Blocker|Should] finding #N from 02-REVIEW.md:
<1-line rationale, if not self-evident>
```

- `type` ∈ {`fix`, `refactor`} for code/config fixes, or `docs` for documentation-only changes that are NOT themselves fixes (e.g. scaffolding, summaries, triage mutation markers).
- **Counting discipline:** the fix-vs-triaged verification uses `^fix(02-rev):` and `^refactor(02-rev):` only. `docs(02-rev):` commits (§1 scaffold, §2 audit capture, §3 triage record, §4 adversarial archival, §5 closure, triage-row mutation markers) do NOT count toward the fix tally.
- One finding = one `fix`/`refactor` commit. No batching per CLAUDE.md §Git.
- Each commit MUST pass CI green before the next is pushed.

**Per-commit verification loop:**
```bash
# Before committing
flutter analyze --fatal-infos --fatal-warnings
dart format --line-length 160 --set-exit-if-changed .
flutter test
dart run tool/check_headers.dart
dart run tool/check_licenses.dart
dart run tool/check_dependencies_md.dart
# All 6 must be exit 0

# Commit + push
git add <files>
git commit -m "..."
git push

# Watch CI
gh run watch
# Only proceed to next finding once conclusion=success
```

**Blocker-waiver forbidden (CONTEXT.md):**
If a user tried to waive a Blocker in Plan 02-02 §3 triage, Plan 02-02 already pushed back. If one slipped through anyway (e.g. reclassified to Should with an override rationale), treat it as Should for commit workflow but flag in 02-04-SUMMARY.md so gsd-verifier can challenge it.

Task 1 mutates §3 triage rows as work progresses (`| fix |` → `| fix (done <hash>) |`) to make progress visible. This mutation breaks any grep that counts `| fix |` rows *during* execution. To preserve a stable ground-truth count, Task 1 writes a snapshot file BEFORE any mutation begins:

- Content: single integer, the count of `^\| [0-9]+ \|.*\| fix \|` rows at start of Task 1
- Lifecycle: written once at start of Task 1, read by verify, deleted at end of Task 2 (phase closure)
