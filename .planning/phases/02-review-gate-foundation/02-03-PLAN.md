**Exit code contract (all 3 gate scripts):**
- `0` = clean (no violation found)
- `1` = policy violation (the case we WANT on adversarial branches)
- `2` = misconfiguration (pubspec.lock missing, .dart_tool absent, script crashed) — INVALID evidence, test must be redone

**CI step ordering inside the `gates` job (`.github/workflows/ci.yml` lines 34-58, critical for header-missing test):**
1. `dart format --line-length 160 --set-exit-if-changed .` (line 37) — will fail first if the poison file is not formatted
2. `flutter analyze --fatal-infos --fatal-warnings` (line 40)
3. `dart run tool/check_headers.dart` (line 43) — the guardrail we want to exercise
4. `dart run tool/check_licenses.dart` (line 46)
5. `dart run tool/check_dependencies_md.dart` (line 49)

For the header-missing test, the poison file MUST be formatted (via `dart format --line-length 160`) BEFORE committing — otherwise the `dart format` step fails first and we never reach `check_headers.dart`. The evidence would then prove the format gate works, not the header gate.
