<objective>
Run the 4 parallel sub-agent audits, synthesize their findings, present them to the user as titles + short explanations (not diffs), and record the user's triage decisions. This is the core review activity of Phase 02.

Purpose: Phase 01 is already `VERIFIED: PASSED` mechanically. Phase 02's value comes from the review layer — catching anti-patterns, subtle bugs, and runtime-only issues that the CI gates cannot see. Splitting the audit across 4 concern-sliced sub-agents (CI gates, bootstrap runtime, code quality, tests+tooling) gives wall-clock parallelism AND specialist coverage that a single Explore agent would dilute.

Output: `02-REVIEW.md` §2 (audit findings, grouped by concern) and §3 (triage decisions) fully populated and committed, plus `02-02-SUMMARY.md` carrying adversarial poison recipes forward to Plan 02-03.
</objective>
