## Decisions Made

1. **Strategy A per-finding atomic commits** -- 15 fix decisions <= 15 single-session threshold per 08.1-RESEARCH §Open Question 3. No relay handoff needed. Contrast: Phase 04 Strategy B 31/10 batches + Phase 08 Strategy A 49/5 relays. For a smaller-scope re-review, per-finding keeps bisect granularity at maximum.
2. **Rows #5 + #6 merged into one commit 5505101** -- both close CountryResolver async-boot races in adjacent listener-wiring code (country_resolver_controller.dart build() + _attachManifestListenerIfNeeded). Commit body enumerates both row numbers + both finding refs (A2#4 + A2#5) for bisect traceability. §3 Commit hash column points both rows at the same SHA.
3. **Row #1 Blocker fix at the UI layer, not at the state-machine layer** -- re-introducing `ErrorState(GpsError)` would reverse the Phase 08 row #37 smell:over-state-machine collapse. The simpler fix keeps the collapse + routes the already-available `AsyncError.error` runtime type through an exhaustive sealed-hierarchy pattern-match in `SessionDetailScreen._handleGpsError`. Controller + state docstrings updated to match.
4. **Row #10 dropped the `isCentering` getter** rather than retaining it for API continuity -- row #36's class collapse was already complete; the getter re-discriminated the sealed variant on a derived boolean. If a third following-semantic lands (e.g. "stale fix"), the future extension is a new field, not another getter. Aligns with CLAUDE.md §is-checks et polymorphisme.
5. **Row #11 resolved as docs-only** -- existing `_viewportSub?.cancel()` guard at `_attachMapViewIfReady` already covers realistic adapter-swap invalidation. Riverpod's instance-per-build semantics mean stale ivars from a disposed instance are unreachable on rebuild. Added build() docstring enumerating the three invariant-preservation mechanisms instead of speculative code change.
6. **Row #13 discrimination via `mounted` flag** -- simpler than introducing a new flag or subscribing to route-change events. Real teardown sees `mounted == false` in the scheduled microtask (nullify proceeds). Kept-alive re-parent sees `mounted == true` across the deactivate/activate pair (nullify skipped). Preserves the row #39 scope-down justification + adds the rotation guard without churning the existing try/catch teardown ordering.
7. **Format-align follow-up as separate `chore(08.1-rev)` commit** -- row #12 Duration hoist nudged SnackBar builder lines past the 160-char limit; dart format re-wraps to canonical form. Phase 08 relay-5 precedent (commit 9ff0286 g.dart format alignment + f95c55c dart-format follow-up on row #7) locked this pattern. Keeping format commits separate from behaviour commits preserves bisect granularity.
8. **STATE.md `completed_phases` counter UNCHANGED at 8** per /gsd:insert-phase decimal escape-hatch. ROADMAP.md §Phase Numbering explicitly reserves decimals for urgent insertions; the integer phase counter tracks the integer phase sequence (01..16), not decimals. `completed_plans` does bump 46 -> 47 because that's a pure tally independent of phase numbering. Rationale documented verbatim in commit body 13bf121 + in the closure Accumulated Decisions entry. Precedent for any future hypothetical 10.1 / 12.1 invocations.
