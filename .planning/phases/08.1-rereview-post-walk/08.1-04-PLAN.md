**Inertness guard canonical reference (Phase 04 Test #3 verbatim, re-used Phase 06 Tests #1-5 + Phase 08 7 tests) :**

Source : `test/infrastructure/db/migration_v1_to_v2_data_loss_test.dart:95-106`

Pattern :
- Pre-assert that the adversary / setup actually ran (precondition that ONLY holds if the test is live)
- THEN the main assertion
- Reason string explains why the test would be inert without this guard

Example :
- `expect(after['t_sessions']! < before['t_sessions']!, isTrue, reason: 'adversarial DELETE did not remove any session row — test would be inert…')`
- `expect(() => sanity.assertNoLoss(before, after), throwsA(...))`

**Mutation experiment protocol (author-time, documented inline in §4 Test block) :**
1. Write test + inertness guard + main assertion
2. Intentionally break the inertness precondition (e.g. make fake FS reject all writes, comment out adversary step, rename required source)
3. Run test — MUST fail LOUDLY with `reason` message (NOT silently pass)
4. Restore — green again
5. Document in §4 Test block : `Mutation experiment (author-time):` paragraph
