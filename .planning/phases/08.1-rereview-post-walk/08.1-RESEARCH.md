### Anti-Patterns to Avoid

- **Re-auditing Phase 07 scope in full:** Phase 08 already did exhaustive 150+ file audit; Phase 08.1 is delta-only. Planner must NOT instruct agents to re-read files the 49-fix loop did not touch AND the walk did not surface.
- **Re-spawning a 5th smell-lens agent:** Rejected Phase 08, stays rejected. Cross-cutting brief + §2 category + §3 tag is the pattern.
- **Fresh smoke before Plan 08.1-02:** The user walk IS the fresh smoke. A second walk post-agents is a fix-loop task, not pre-engaged.
- **Adversarial branch without a new gate script:** Breaks the 1-branch-per-gate-script convention (Phase 02+04+06+08 all held).
- **Copying 08-CONTEXT verbatim without narrowing:** Phase 08.1 is explicitly "narrowed scope to delta". A verbatim copy would double the wall-clock cost and produce redundant findings.
- **Waiting to capture walk findings until after agents spawn:** Violates user-first protocol. §1b walk-evidence is blocking for §2/§3/§4.

## Common Pitfalls

### Pitfall 1: Walk-Findings Not Yet Captured at Planning Time

**What goes wrong:** Plan 08.1-02 tries to extract walk evidence but `docs/phase-08.1-walk.md` doesn't exist because the user hasn't run the walk yet.

**Why it happens:** `/gsd:plan-phase 08.1` may run BEFORE the user does the Android + iOS walks. The walks take 2-5 min each + screenshots + observations — not zero wall-clock.

**How to avoid:** Plan 08.1-02 MUST include a `checkpoint:human-verify` gate at the top: "User has run Android walk + iOS walk + committed `docs/phase-08.1-walk.md` + screenshots under `docs/phase-08.1-walk-screenshots/`". Until that gate clears, no agent spawns.

**Warning signs:** Empty or missing `docs/phase-08.1-walk.md` at Plan 08.1-02 execution; user message "I haven't walked yet" post-`/gsd:plan-phase`; git log shows no `docs(08.1-walk): ...` commit.

**Recovery:** Pause at the checkpoint ; surface the gap to the user explicitly ; suggest walk protocol (re-use Phase 07 `docs/phase-07-smoke.md` steps as template — install APK + IPA, cold start, navigate /map, trigger airplane mode, trigger download, trigger delete, observe + screenshot).

### Pitfall 2: Over-Scoping the Agents — Re-Auditing All Phase 07

**What goes wrong:** Agent #1 gets briefed "audit `lib/infrastructure/map/`" and re-reads 40+ files that Phase 08 already cleared + the 49-fix loop didn't touch + the walk didn't flag. Produces 50+ redundant findings.

**Why it happens:** Phase 08 brief template describes layer + risk in terms of the whole Phase 07 scope. Direct-copy without narrowing = re-audit.

**How to avoid:** Every agent's brief MUST explicitly list the files they cover, derived from `git log --grep='08-rev' --name-only` intersected with the walk findings + 9 smell-refactored components. Agent prompts for 08.1 should have an explicit "files NOT in scope" negation section.

**Warning signs:** Agent brief mentions files that haven't changed since Phase 08 close commit `254b5d2`. Agent's findings include items identical to Phase 08 `accepted-as-is` rows (rows 51, 60-69, 71-75).

**Recovery:** If detected at agent-spawn time, re-brief with explicit file list. If detected post-synthesis, triage as `accepted-as-is` and document cross-reference to Phase 08 row.

### Pitfall 3: Adversarial Branch Created Without a New Gate

**What goes wrong:** Plan 08.1-04 spawns `adversarial/08.1-*` branch but there's no NEW CI gate script to validate. Branch pushes + CI runs but has nothing actionable to assert; becomes ceremonial.

**Why it happens:** Phase 08 had a new gate (`check_style_no_external_url`) so it had a branch. Inertia copies the pattern without checking the precondition.

**How to avoid:** Plan 08.1-04 is **SKIP by default**. Explicit condition check: "does walk surface a policy needing a new CI gate?" If no, Plan 08.1-04 is a small no-op wave that verifies the existing gates still green + the Phase 08 branch still deleted.

**Warning signs:** Plan 08.1-04 plan frontmatter mentions an `adversarial/08.1-*` branch without a matching `tool/check_*.dart` NEW script.

**Recovery:** If planned, revert: delete the adversarial plan task. Replace with permanent regression test(s) if walk demanded one.

### Pitfall 4: Smell-Heuristics Re-Check Turns Into Re-Refactor

**What goes wrong:** Agent #1/#2 identifies "row #20 DownloadState dispatcher collapse could further simplify..." and flags as Blocker. Phase 08.1 fix-loop attempts a second-level rewrite of an already-refactored component.

**Why it happens:** Smell-heuristics lens is subjective ; a landed refactor can always be further-refactored. Without discipline, 08.1 spirals into infinite-simplification.

**How to avoid:** The smell-heuristics brief for 08.1 agents MUST include: "Phase 08 already did one smell-pass on these 9 components. Your lens is: did the Phase 08 refactor INTRODUCE a new smell? Is there a regression? Is there a walk-findings-aggravated shape? DO NOT propose further simplification without evidence of a concrete pain point (bug, test failure, walk-observation)."

**Warning signs:** Agent findings tagged `smell` without a referenced bug / regression / walk-observation. Triage row with `refactor` decision but no concrete rationale beyond "could be simpler".

**Recovery:** At §3 triage, downgrade evidence-less `smell` findings to `Could` / `accepted-as-is`. User triage signal.

### Pitfall 5: Session Context Budget Explosion

**What goes wrong:** The full Phase 08 procedure (5 plans × per-finding atomic commits × CI waits) took 5 session relays and ~5h wall-clock for 49 findings. Phase 08.1's delta could still be 15-30 findings; single session might exhaust.

**Why it happens:** Context accumulates across plan scaffolds + agent outputs + triage tables + fix commits.

**How to avoid:** Apply the Phase 08 relay pattern proactively. `.fixes-expected` file landed at Plan 08.1-04 closure ; §3 `(pending)` markers in the triage table. Orchestrator expects relay handoff if row count > 20.

**Warning signs:** Single fix commit taking > 15 min of session time ; repeated "context budget" warnings ; agent rate-limiting.

**Recovery:** Split Plan 08.1-05 across relays at natural CI-green boundaries. Each relay picks up from the first `(pending)` row ; applies fixes ; commits ; CI-gates ; signs off.

### Pitfall 6: Row Numbering Collision with Phase 08

**What goes wrong:** Phase 08 §3 has rows 1-75. Phase 08.1 §3 starts fresh but uses numbers 1-30 — confusing during cross-reference in commit messages.

**Why it happens:** Both review gates use the same `#N` convention.

**How to avoid:** Commit prefix `fix(08.1-rev):` disambiguates. Also, §3 header docstring should note "Row # is local to Phase 08.1 §3 ; Phase 08 §3 rows prefixed `08§3-#N` when cross-referenced".

**Warning signs:** A commit message says `fix(08.1-rev): ... (row #5)` while also cross-referencing Phase 08 row #5 — ambiguous.

**Recovery:** Adopt an explicit cross-reference format in 08.1 §3 rationale cells when citing Phase 08: `cross-ref 08§3-#N` or `Phase 08 row #N`.
