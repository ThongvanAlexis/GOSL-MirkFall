**Decision on JSON timestamp shape for export (RESEARCH ambiguous):**

CONTEXT.md §DateTime says JSON export is a single ISO 8601 field: `"startedAt": "2026-06-10T08:00:00+02:00"`. That needs a custom Freezed `fromJson`/`toJson` converter because the DOMAIN carries `(startedAtUtc, startedAtOffsetMinutes)` split — one field in JSON → two fields in memory.

Pattern: Add `@JsonKey(fromJson: _startedAtFromJson, toJson: _startedAtToJson, name: 'startedAt')` on `startedAtUtc` and mark `startedAtOffsetMinutes` with `@JsonKey(includeFromJson: false, includeToJson: false)`. Implement the two free functions in the same file that parse `"2026-06-10T08:00:00+02:00"` into `DateTime` (UTC) and the reciprocal encode. For read-side, the offset is recovered from `DateTime.parse(s).timeZoneOffset.inMinutes` before `.toUtc()`.

Catch: `DateTime.parse` on `"2026-06-10T08:00:00+02:00"` returns a NON-UTC DateTime with `timeZoneOffset = 2h`. `parsed.toUtc()` gives the UTC equivalent; `parsed.timeZoneOffset.inMinutes` captures offset. Write the codec in a tiny helper file `lib/domain/sessions/session_timestamp_codec.dart` (top-level functions, no class) or inline as `static` methods on `Session` — implementer's choice. Test both orderings in `session_timezone_test.dart`.

For 03-03 simplicity, if the Freezed / json_serializable combo makes the split-then-rejoin awkward, a valid fallback: **ship JSON with TWO fields** (`"startedAtUtcMs"` + `"startedAtOffsetMinutes"`) for Phase 03. Note in SUMMARY.md that the single-ISO-string export will be finalized in Phase 13 (PORT-02 readability gate). SC#5 is satisfied either way — the JsonMigrator doesn't care about timestamp field shape.

**Recommended path: ship the split-fields JSON for Phase 03, defer the ISO-combined form to Phase 13.** It reduces Phase 03 risk (no custom json_serializable converter), and Phase 13 already owns the `SCHEMA.md` formalization.

/// Renderer-specific configuration attached to a [MirkStyle].
///
/// Sealed union — exhaustive switch at the render call site (Phase 09).
/// Unknown `rendererType` values (imports from future versions) fall through
/// to [UnknownConfig] carrying the raw map, preserving data survival across
/// version skew without requiring an app upgrade.
@Freezed(unionKey: 'rendererType', fallbackUnion: 'unknown')
sealed class MirkStyleConfig with _$MirkStyleConfig {
  const factory MirkStyleConfig.atmospheric({
    @Default(0xFF000000) int baseColorArgb,
    @Default(0.5) double noiseScale,
  }) = AtmosphericConfig;

  const factory MirkStyleConfig.shader({
    required String shaderAssetPath,
  }) = ShaderConfig;

  const factory MirkStyleConfig.unknown({
    required Map&lt;String, Object?&gt; raw,
  }) = UnknownConfig;

  factory MirkStyleConfig.fromJson(Map&lt;String, Object?&gt; json) =&gt;
      _$MirkStyleConfigFromJson(json);
}

/// Port for session persistence + lifecycle.
///
/// Implementations live in `lib/infrastructure/stores/` (03-06).
abstract class SessionStore {
  /// Returns all sessions (active + stopped), ordered by [Session.startedAtUtc]
  /// descending by default.
  Future&lt;List&lt;Session&gt;&gt; listAll();

  /// Returns the session with [id] or null if none exists (find semantic).
  Future&lt;Session?&gt; findById(SessionId id);

  /// Returns the session with [id] or throws [SessionNotFoundException]
  /// (require semantic — caller promises it exists).
  Future&lt;Session&gt; requireById(SessionId id);

  /// Returns the single active session, or null if none is active.
  /// Exclusivity is enforced at the DB layer (SESS-06); this method never
  /// returns more than one.
  Future&lt;Session?&gt; findActive();

  Future&lt;void&gt; insert(Session session);

  Future&lt;void&gt; update(Session session);

  /// Hard-deletes the session and cascades: markers, revealed_tiles, and
  /// photos attached to those markers are removed in the same transaction
  /// via FK ON DELETE CASCADE.
  Future&lt;void&gt; delete(SessionId id);

  /// Activates the given session.
  ///
  /// SESS-06 exclusivity is enforced via a partial unique index on
  /// `t_sessions(status='active')` — a second concurrent activation from
  /// another code path throws [ConcurrentActivationException].
  Future&lt;void&gt; activate(SessionId id);

  /// Stops the session (transitions status `active → stopped`).
  Future&lt;void&gt; deactivate(SessionId id);
}

**Step 1: Ship `lib/domain/envelope/envelope.dart` as Freezed (Blocker 1 fix).**

ROADMAP SC#4 verbatim requires `Envelope` to be Freezed. Freezed + json_serializable with a `Map<String, Object?> payload` field requires a custom `@JsonKey` converter. The cleanest pattern: mark the field with `@JsonKey(fromJson: _payloadFromJson, toJson: _payloadToJson)` and write two top-level helper functions that cast the raw map directly.

```dart
// Copyright (c) 2026 THONGVAN Alexis
// Licensed under the Good Old Software License v1.0
// See LICENSE file for details
import 'package:freezed_annotation/freezed_annotation.dart';

import '../errors/import_errors.dart';

part 'envelope.freezed.dart';
part 'envelope.g.dart';

/// Versioned JSON envelope for import/export (decision D9).
///
/// Shape: `{ schemaVersion: int, type: string, payload: {...} }`. Every
/// shareable document wraps its body in this envelope so [JsonMigrator] can
/// read [schemaVersion] and dispatch migrations before handing [payload] to
/// the entity factory.
@freezed
abstract class Envelope with _$Envelope {
  const factory Envelope({
    required int schemaVersion,
    required String type,
    @JsonKey(fromJson: _payloadFromJson, toJson: _payloadToJson)
    required Map<String, Object?> payload,
  }) = _Envelope;

  /// Parses [json] into an [Envelope]. Throws [ImportValidationException] on
  /// missing or malformed required keys — delegated to manual pre-validation
  /// before Freezed's generated fromJson runs so the domain-owned
  /// [ImportValidationException] surfaces instead of `CheckedFromJsonException`.
  factory Envelope.fromJson(Map<String, Object?> json) {
    final rawVersion = json['schemaVersion'];
    if (rawVersion is! int) {
      throw ImportValidationException(
        reason: 'envelope: schemaVersion must be an int, got ${rawVersion?.runtimeType}',
      );
    }
    final rawType = json['type'];
    if (rawType is! String || rawType.isEmpty) {
      throw ImportValidationException(
        reason: 'envelope: type must be a non-empty string',
      );
    }
    final rawPayload = json['payload'];
    if (rawPayload is! Map) {
      throw ImportValidationException(
        reason: 'envelope: payload must be an object, got ${rawPayload?.runtimeType}',
      );
    }
    return _$EnvelopeFromJson(json);
  }
}

Map<String, Object?> _payloadFromJson(Map<String, dynamic> raw) =>
    Map<String, Object?>.from(raw);

Map<String, dynamic> _payloadToJson(Map<String, Object?> payload) =>
    Map<String, dynamic>.from(payload);
```
