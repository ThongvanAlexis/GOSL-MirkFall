## Decisions Made

See `key-decisions` in frontmatter for the full list. Most load-bearing:

1. **Pubspec swap as the very first commit** — all downstream work needs `maplibre_gl` resolvable, so the dep graph change is atomic at commit #1 and every subsequent commit operates against the new lockfile.
2. **Bbox polygon simplification (not mapshaper)** — pure-Dart one-shot script emits axis-aligned bounding boxes per country. 176 KB total for 249 countries, well under the 5 MB budget. Acknowledged tradeoffs (overlap at shared latitudes, enclaves) with tie-breaking protocol documented in both the script and Plan 07-03's consumer path.
3. **`tool/prepare_style.dart` ships in placeholder mode** — real Protomaps basemaps-assets clone requires git + network + repo-write during a maintenance window, and Plan 07-03 first-launch copier does not need real glyphs at z0-2 (the world bundle renders acceptably without text labels at those zoom levels). Real assets land alongside the first MapLibre render (Plan 07-06).
4. **`ios/Podfile.lock` kept as Phase 01 placeholder** — the tree already has the Windows-dev comment-only placeholder; CI's Windows-dev-bootstrap step handles placeholder detection + removal before `pod install`. Deleting it would re-introduce the Phase 01 resolution trace for no gain.
5. **`mirk_fog` layer = `type: background` with `background-opacity: 0`** — a `fill` layer would require a bound empty-GeoJSON source to paint nothing, which is wasteful. Phase 09 replaces this with `type: fill` + a real MirkRenderer source at the same z-index position.

## Issues Encountered

1. **`package:convert` not a direct dependency.** First draft of `tool/generate_world_sha256.dart` used `AccumulatorSink` + `hex.encode` from `package:convert`, which is only transitively available via `crypto`. Rather than promote `convert` to a direct dev dep (not listed in Plan 07-01 Task 1), simplified the script to `readAsBytesSync` + `Digest.toString()` (crypto emits hex by default). The 856 KB asset is small enough that the in-memory read is trivial; the streaming shape still lives in the docstring for Plan 07-04 to copy when it handles 1.5 GB chunks.
2. **`flutter_local_notifications 21.0.0` dropped its `intl` transitive.** Plan said "if `intl` survives (retained by flutter_local_notifications), keep its row" — it did not. Cleanly dropped with the other flutter_map transitives. Zero impact.
3. **`xml 6.6.1` re-anchored to `flutter_local_notifications` (via `timezone`)** after `flutter_map` was dropped. Updated its DEPENDENCIES.md "Pulled in by" column to reflect the new parent. No behaviour change.
4. **Linter warnings on `tool/prepare_style.dart` for angle-bracket doc comments (unintended HTML)** and `tool/simplify_polygons.dart` for `prefer_interpolation_to_compose_strings`. Both fixed immediately; scripts now pass `flutter analyze --fatal-infos --fatal-warnings`.
